Changelog

Notable changes to pig, newest first.

v0.1.2 2026-03-07 patch
Interactive session
  • fix interactive session wiring was missing from the published bundle in v0.1.1; pig with no arguments now correctly launches the grooming session
v0.1.1 2026-03-07 patch
Interactive session
  • fix pig with no arguments now launches the interactive grooming session instead of printing help and exiting
  • new wired runPackageListSession to showPackageDetail as the per-package picker, connecting the list TUI to the full strategy/detail flow
  • new applied changes are now written to disk via the real updatePackageJsonRange, runPnpmInstall, and runValidationHook implementations
Docs
  • Updated homepage and changelog favicon and logo to full-body pig 🐖
v0.1.0 2026-03-04 initial release
Commands
  • new pig — interactive grooming session with per-package strategy selection, change queue, and apply/discard confirmation
  • new pig scan — read-only audit; scans all configured paths and reports version conflicts without modifying any files
  • new pig store — prints the pnpm content-addressable store path, total size, and file count
Strategies
  • new lowest — pin the lowest semver version currently in use across all projects
  • new most-acceptable — pick the version whose range is already satisfied by the most projects; prompts to break ties
  • new project — use the version installed in a specific project you choose
  • new latest — fetch dist-tags.latest from the npm registry
  • new specific — browse all published versions from the registry and pick one interactively
Apply pipeline
  • new package.json range updater with semver-aware transformation — ^/~ only update when the range doesn't already satisfy the target; >=/<= always update; workspace: and * are never touched
  • new validation hook — runs a configurable shell command after each pnpm install; reverts both package.json and pnpm-lock.yaml atomically on non-zero exit
  • new dry-run mode via --dry-run — prints proposed changes without writing files or running installs
Configuration
  • new global config at ~/.pig/config.json — scanPaths, ignore, validate, registryUrl
  • new per-project override at .pig.json — overrides the global validate hook for that project only
  • new --package flag — focus the session on a single named package
  • new --strategy flag — apply a named strategy to all packages without interactive prompting
  • new --paths flag — override scanPaths for a single run
  • new --validate / --no-validate flags — override or skip the validation hook for a session
Lockfile support
  • new pnpm lockfile v6 parser (pnpm 7/8)
  • new pnpm lockfile v9 parser (pnpm 9/10)
